So far, I feel like I'm doing good. It's very complicated setting it all up, but it's worth it to see it all come together.

Thew way my site is set up is as follows:

  • The root folder contains an index.html page to direct the user towards the different versions of the site.
  • Within the root folder, there is:
    • Root folders of the websites themselves
    • /static/, which contains the building blocks of the website
    • /tools/ which contains scripts used to generate the static website
  • Within /static/, there is:
    • /fragments/, which contains HTML snippets that get combined into webpages
    • /images/,
    • /posts/, which contains blog articles
    • /templates/, which contains boilerplates of pages
  • How do I work on the the website?
    • To ppublish to the blog: Write a Blog post in /static/posts with a filename in the format "YYYY-MM-DD-Title" and run the refresh blog script
    • To add a project, write the inner code in /static/projects and generate the full page with the script.
    • The website updates when i commit to it
    • The projects index and projects section on the homepage is hardcoded
  • What's my next task?
    • Make the templates for the index and project pages
    • Make page generators for the templates
    • Start writing up webpages for the projects
